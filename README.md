# PolyGuess - Issues & Feedback

This repository is for **bug reports, feature requests, and feedback** for [PolyGuess](https://polyguess.ca), a daily prediction game.

> **Note**: This is an issues-only repository. No code is hosted here. For the actual game, visit [polyguess.ca](https://polyguess.ca).

## What is PolyGuess?

PolyGuess is a free-to-play daily prediction game where you guess the probability of real-world events sourced from Polymarket. Test your calibration skills and compete against other players!

## Reporting Bugs

Found a bug? Please [open an issue](https://github.com/austinbrodeur/polyguess-issues/issues/new) with:

- **Description**: Clear description of the bug
- **Steps to Reproduce**: How can we reproduce the issue?
- **Expected Behavior**: What should happen?
- **Actual Behavior**: What actually happens?
- **Screenshots**: If applicable, add screenshots
- **Browser/Device**: Your browser version and device type

### Example Bug Report

```
Title: Slider not working on mobile Safari

Description:
The probability slider doesn't respond to touch on iPhone Safari (iOS 17.4).

Steps to Reproduce:
1. Open PolyGuess on iPhone Safari
2. Try to drag the slider
3. Nothing happens

Expected: Slider should respond to touch
Actual: Slider is unresponsive

Browser: Safari 17.4 on iPhone 14 Pro
```

## Feature Requests

Have an idea to make PolyGuess better? We'd love to hear it!

Please [open an issue](https://github.com/austinbrodeur/polyguess-issues/issues/new) with:

- **Feature Description**: What feature would you like to see?
- **Use Case**: Why would this be useful?
- **Proposed Solution**: (Optional) How might this work?

### Example Feature Request

```
Title: Add leaderboard for weekly scores

Description:
It would be cool to see a leaderboard showing the top players each week.

Use Case:
- Adds competitive element
- Motivates players to return daily
- Community engagement

Proposed Solution:
Could be a new page showing:
- Top 10 players for current week
- Weekly averages
- Maybe reset on Sundays?
```

## Security Issues

**Do NOT report security vulnerabilities publicly!**

If you discover a security issue, please email me directly (contact info in my GitHub profile) instead of opening a public issue.

## Please Don't

- Request features that compromise privacy (e.g., user accounts, personal data collection)
- Report issues with Polymarket itself (we just use their public API)
- Post spam, advertisements, or unrelated content
- Be rude or disrespectful to others
